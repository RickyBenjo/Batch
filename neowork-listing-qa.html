8-Step QA Workflow

Before you start

Only work from the NeoWork tab in Vendor Directory → Recently Live. Basic & Core partners only. Boost → Aislinn. Party Pro → Rachel.

1
Open Vendor Directory → Recently Live → NeoWork tab Start from the bottom of the list (oldest listing first). These are listings that went live but haven't been reviewed yet — some go back to March 24.
2
Click the listing → Assign your name (no checkmark yet) Add yourself as the assignee. Do NOT check the done box until every step is complete.
No checkmark until fully done
3
Run the full QA Checklist — Listing Details tab Title casing, booking URL with UTM, duration, location, address, description, all media. Use the Checklist tab above to track every item.
Open every link to verifyWatch full videos — no skipping or muting
4
Check Attachments, Ratings & FAQs Menus/maps as PDFs in Attachments (not media). At least 10 ratings. FAQs copied from partner's website, one Q+A per block — only if they have them.
5
Check Admin tab — Category & Sub-category tags Verify the correct category and sub-category are applied. Use the Categories tab for the full reference. Check similar live listings when unsure.
6
Check Partner Profile — fill in any missing info Logo (required), social links (Facebook ID only — not the full URL), main website, public phone number (only if officially publicly listed).
Completing the profile = better SEO indexing
7
Optimize the listing — tags + About section Add/refine tags and improve the About This Experience section using CoStar and other tools. Same process as previous optimization work.
8
Send partner email → then add checkmark Let them know you optimized their listing and highlight what they can do next (ratings, videos, social links). Focus on adding value — not performance data. Use the Email Generator tab. Only after sending: add the checkmark.

Partner Status Definitions

Onboarding First live date is 30 days or less
Active First live date is more than 30 days ago

Beth's Role (separate queue)

High-level audit only: checks for duplicate listings, merge opportunities, and overall structure. Beth reviews from a bird's-eye view and does not handle individual QA checkboxes.

Queue Organization

There should be separate queues for NeoWork, Beth, Boost, and Party Pro. The Recently Live interface has sub-sections to toggle between them. Once the final QA checkbox is selected, the listing is removed from the queue.
